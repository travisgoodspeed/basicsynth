Project Files

A BasicSynth project file is an XML format file that contains general information about the composition, synthesizer settings, instrument definitions, score files, and parameters that control the format of the output. Generally, project files are not edited by hand since all entries in the file can be set from the program. An template project file is shown below. 

<synthprj>
  <name>Composition name</name>
  <author>Composer name</author>
  <desc>Description of the composition</desc>
  <cpyrgt>Copyright notice</cpyrgt>
  <wvdir>wave file path</wvdir>
  <libdir>path to find libraries</libdir>
  <synth sr="44100" fmt="0" wt="16384" usr="0" >
    <wvtable ndx="" id="" parts="" gibbs="" type="" >
      <part mul="" amp="" phs=""/>
    </wvtable>
  </synth>
  <midi>
    <chnl cn="" bnk="" prg="" vol="" pan="" />
  </midi>
  <out type="1" lead="" tail="">
    output file path
  </out>
  <wvfile id="" name="" desc="" >file name</wvfile>
  <mixer chnls="" fxunits="" lft="" rgt="">
    <chnl cn="" on="" vol="" pan=""/>
    <reverb unit="" vol="" rvt="" pan="" >
      <send chnl="" amt=""/>
    </reverb>
    <flanger unit="" lvl="" mix="" fb="" 
      cntr="" depth="" sweep="" pan="" >
      <send chnl="" amt="" />
    </flanger>
    <echo unit="" tm="" dec="" >
      <send chnl="" amt="" />
    </echo>
  </mixer>
  <libfile>instrument library file</libfile>
  <sndbnk name="" inc="1" pre="">
    path to sound bank (SF2/DLS)
  </sndbnk>
  <instrlib>
    <instr> instrument definition (specific to each instrument)</instr>
  </instrlib>
  <seq name="">a sequencer file</seq>
  <score name="" dbg="">a notelist file</score>
  <text>file associated with the project</text>
</synthprj>
Tag Attribute Use
synthprj   Document root tag
name   Name of the composition
author   Composer name
desc   Description
cpyrgt   Copyright notice
out   Output file path
  type 1 = WAVE file, 2 channel. (This is the only file type current supported)
  fmt Data format, 0=16-bit PCM, 1=32-bit float.
  lead Seconds of silence at beginning of the file
  tail Seconds of silence at end of file. When reverb is enabled, this should be at least as long as RVT.
libdir   Path to libraries
wvdir   Path to wave files
synth sr Sample rate (default 44100)
  wt Wavetable length (default 16k. Values greater than 16k may cause problems.)
  usr Number of user defined wave tables, defined by wvtable tags
wvtable ndx Wavetable index number (0-9 are reserved for library use)
  id Wavetable ID, same as index for pre-defined tables.
  parts Number of partials
  gibbs Apply adjustment for Gibbs effect, 1=apply, 0=don't apply (only used for sum of sin waves)
  type Wave type, 1=sum of sines, 2=linear segments
part mul Partial number (1-n) (not used for linear segments)
  amp Amplitude for this partial/segment
  phs Starting phase of this partial, in radians (optional), or segment duration
mixer chnls Number of mixer inputs
  fxunits Number of effects units
  lft Master volume left
  rgt Master volume right
chnl cn Mixer input channel number
  on Channel on (1) or off (0)
  pan Channel pan (-1 to +1)
reverb unit Effects unit number
  vol Output volume level
  rvt Reverb time
  pan Pan for the reverb
send chnl Input channel to send from
  amt Amplitude to send to effects unit
flanger unit Effects unit number
  fb Flanger feedback amount
  center Flanger center delay (seconds)
  depth Flanger delay sweep depth (seconds)
  lvl Flanger output level
  mix Mix of input to flanger output
  sweep Sweep rate in Hz
  pan Pan position for flanger output
echo unit Effects unit number
  tm Echo delay time
  dec Echo decay value
wvfile   Wave file to load
  id ID to associate with the file
  name Display name
  desc Description
libfile   Instrument library
instrlib   In-line instrument library
instr * See the definition of each instrument in the Instrument Library
seq   A sequencer events file (See Sequencer Files)
  name Display name for the file
score   A Notelist file (See Notelist)
  name Display name for the file
text   File associated with the project

The name, author, desc, copyright, synth, mixer, wvdir and out tags should appear once. The other tags can be repeated as many times as needed.

The synthesizer parameters must be set prior to loading instruments, and instruments must be loaded prior to processing score or sequence files. The BSynth program insures the proper order by making three scans of the project file. The first scan looks for synth and mixer nodes. The second scan loads instruments and instrument libraries. The third scan processes score and sequence files.

Wave Tables

The synthesizer defines a set of common wavetables automatically. To include additional tables, the usr attribute of the synthesizer node must indicate how many user wave tables should be allocated. Each wave table is defined in a wvtable node.

The wavetable is identifed by index and ID values. The index value indicates a specific position in the wavetable array maintained by the wtSet object. The first 10 entries are pre-defined, and, although they can be reloaded with user defined values, they should typically be left as-is. User wavetables thus begin with index 10. The wavetable oscillators use the wave table ID, but since the predefined entries have ID values equal to the index, either the index or ID may be used. The index and ID values can be the same for user tables as well. However, when wavetables are defined in an instrument library file, the ID should be indicated and used in order to avoid number conflicts with other libraries.

User wave tables defined in the project file may be one of two types. Type 1 wavetables are constructed as a series of sin waves and are typically used as audio waveforms. The amp, mul and phs attributes on each partial represent the amplitude, frequency multiple and initial phase of each sin. Note that the mul values must be positive integers. The phs values are in radians. Amplitude values can be in any convenient range and represent relative amplitude levels. The wavetable will be rescaled so that the peak amplitude is normalized to [-1,+1]

Type 2 wavetables are defined as a series of line segments and are typically used for LFO and pitch bend modulation effects. The amp attribute indicates the end amplitude of the segment (normalized to +/-1.0), while the phs attribute indicates the duration of the segment specified as a fraction of the wavetable length. For example, a part with phs=0.5 would create a segment equal to half the wavetable length. The starting value for the wavetable is 0. To start the wavetable at some other value, the first segment should indicate the starting amplitude value and use a duration (phs) of 0.

Wave Files

The WFSynth instrument is used to playback pre-recorded sounds stored in wave files. Each wave file must be included in the project using a wvfile node. Wave files are typically 2-channel, PCM format (16-bit at 44100k sample rate) but 32-bit float formats can be used as well. Sample rates are automatically converted to the synthesizer sample rate by either skipping or duplicating samples as needed. However, it is better to use a sound file editor to convert the file sample rate to the synthesizer sample rate since the sound file editor can apply more sophisticated sample rate conversion algorithms.

Since wave files may be located in a shared directory, the project may also specify the path to wave files using the wvdir node. A default path for all projects may also be specified in the project options (Project->Options on the main menu.)

Instrument Library Files

Typically, the instruments used in a composition are defined in the project file. However, instruments that are used by multiple compositions can be collected into an instrument library file and then incorporated by reference using the libfile tag. An instrument library is an XML file with a root node of instrlib and child nodes with tag instr. In addition, wavefile and wavetable nodes can be embedded into the library. Wavefiles should be given ID values that will not overlap other instrument definitions. Likewise, wavetables should be given ID attibute values that are unique from other instrument libraries.

Instruments located in the library can be given ID values that are unique, or can be given the same ID (e.g., 0) and the instrument manager will automatically assign a unique number to the instrument. It is also good practice to create a unique prefix for library instrument names so that the names do not duplicate instrument names from othe library or project files.
